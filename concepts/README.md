# Concepts

This directory contains explanations of the core concepts that make up modern system design.

Each concept is explained from **first principles**—starting with the problem, understanding why it exists, exploring how it works, and finally seeing how it fits into a real-world distributed system.

The goal is to build intuition rather than memorize definitions.

## Learning Approach

Each concept follows the same structure:

1. **The Problem** – What challenge does this solve?
2. **The Solution** – How does it solve the problem?
3. **How It Works** – The internal mechanics.
4. **Real-World Example** – Applying the concept to systems like Uber or Netflix.
5. **Advantages & Trade-offs** – Benefits and limitations.
6. **Interview Questions** – Common questions and answers.

Understanding the problem first makes the solution easier to remember and apply.
